如何将create-react-app项目部署到Github Pages

· 2020-04-20 09:49:43 · # React # Github

昨天写了个项目，使用create-react-app搭建的，在本地能跑通。我还想给这个项目部署上去，让我随时随地能访问。

前提条件

  1. 项目使用 create-react-app 构建；
  2. react-scripts 版本需在 0.2.0 或以上；
  3. 项目在本地 yarn start 或者 npm start 能跑通；
  4. 在 GitHub 远程存在一个与这个本地项目对应的代码仓库。

部署流程

  1. 删除或者注释掉 .gitgnore 文件中的 /build.
# production
/build     // 把这行注释掉或者直接删除
  1. 在 package.json 中加入 "homepage": "./", .
{
  "name": "cool-nav",
  "version": "0.1.0",
  "private": true,
  "homepage": "./",   // 添加这一行
  ……
}
  1. 安装 gh-pages 包.
yarn add gh-pages --save-dev
  1. 在 package.json 中的 scripts 配置中添加配置项："deploy": "gh-pages -d build".
{
    ……,
    "scripts": {
        "start": "react-app-rewired start",
        "build": "react-app-rewired build",
        "test": "react-app-rewired test",
        "eject": "react-scripts eject",
        "deploy": "gh-pages -d build"    // 新增这一行
    },
    ……
}
  1. 构建项目：在命令行中输入 yarn build 构建项目.
yarn build
  1. 发布项目：构建成功后，使用 yarn deploy 命令将项目发布到GitHub Pages上.
yarn deploy

发布之后，gh-pages 会在远程创建一个 gh-pages 分支，并且将编译出来的 build 文件夹发布到该分支上。

  1. 确认分支：在该项目的GitHub远程库中是否有 gh-pages 分支，并确认该分支的代码是否是 build 文件夹下的内容。
  1. 确认发布成功：找到该项目的Setting => Options => GitHub Pages，看到这样的绿条就说明构建成功。
  1. 访问：发布成功后，需要静待一段时间才能访问到最新的代码。

注意事项

每次发布之前都需要先执行 yarn build 编译项目后，再执行 yarn deploy 将项目发布上去。
